## Design Context

### Users
Vitsoe customers planning their own 606 modular shelving system before purchasing. Design-conscious consumers who already appreciate Vitsoe's aesthetic and Dieter Rams' philosophy. They need to visualize configurations confidently and understand pricing — the tool should feel like an extension of the Vitsoe experience, not a generic web app.

### Brand Personality
**Honest, unobtrusive, purposeful.**

Inspired by Dieter Rams' 10 principles for good design — particularly "as little design as possible," "honest," and "unobtrusive." The interface should disappear into the task. Every element earns its place. No decoration for decoration's sake.

### Aesthetic Direction
- **Visual tone**: Ultra-functional minimalism with warmth. Not cold or sterile — quiet confidence. The UI should feel like a well-designed tool, not a marketing page.
- **Theme**: Light only. Clean, bright, airy — consistent with the 606 system's visual identity of open, breathing space.
- **Primary reference**: vitsoe.com — restrained, black-and-white dominant, generous whitespace, functional hierarchy
- **Other references**: Dieter Rams' design principles, Braun product design, the 606 system itself (aluminum, wood, honest materials)
- **Anti-references**: Generic SaaS dashboards, purple-gradient AI aesthetic, over-decorated skeuomorphic tools, anything "techy" or startup-flavored

### Design Principles

1. **Less but better** — Every UI element must earn its place. Remove before adding. Reduce to the essential.
2. **Honest materials** — Show real information, use real proportions. No fake depth, no gratuitous effects. If something is a button, it looks like a button.
3. **The tool disappears** — The interface should never compete with the user's configuration. The 606 system is the star; the UI is its quiet frame.
4. **Precision over flair** — Alignment, spacing, and typography should be meticulous. Quality is in the details, not in flashy features.
5. **Accessible by default** — WCAG AA compliance. 4.5:1 contrast minimum for text, keyboard navigable, respects reduced motion preferences.

### Typography (derived from vitsoe.com)
- **Brand typeface**: Univers (Adrian Frutiger, 1969) — core to Vitsoe identity, chosen by Wolfgang Schmidt
- **Web equivalent**: Since Univers requires licensing, use a free geometric-humanist sans-serif that captures the same spirit. Candidates: **DM Sans** (warm, geometric, good weight range) or **Source Sans 3** (humanist, functional, Univers-adjacent)
- **Hierarchy approach on vitsoe.com**: Bold 700 for headings, regular for body. Sizes range from 14px (secondary) to 32px (headings) with generous line-heights (e.g. 32/40 on headings, 17/24 on body)
- **Scale**: Fixed `rem` sizing for app UI — not fluid
- **Details**: `tabular-nums` for pricing, restrained uppercase/letter-spacing for section labels only
- **Readability**: Body text 16px+, comfortable line-height (1.4-1.5 for body)

### Spacing System (derived from vitsoe.com)
Base unit: 8px. Common values: 8, 16, 24, 32, 40, 48, 64px.
- Mobile gutters: 16px
- Desktop gutters: 32px
- Section spacing: 40-64px
- Max content width: 1110px

### Layout
- Mobile-first responsive (breakpoints at 620px, 860px)
- Flexbox and grid-based
- Generous whitespace — space is a design element, not waste
- Borders: 1px solid `#c2c2c2` — clean, functional separators
